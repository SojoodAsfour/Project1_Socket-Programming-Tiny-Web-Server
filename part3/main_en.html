Summary: HTTP/1.1 caching (RFC 2616) allows clients and intermediaries to store responses so they can be reused without contacting the server every time. Caching behavior is controlled mainly by headers like Cache-Control, Expires, ETag, Last-Modified, and Age. These headers determine how long a response stays fresh and when it must be revalidated with the server. The goal is to improve performance, reduce bandwidth, and keep content up-to-date.
